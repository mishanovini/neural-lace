## Notes & limitations

- **STALE is the priority class** — it means a doc tells contractors to use UI that no longer exists. Exit code is 1 when any STALE is found.
- UNDOCUMENTED is best-effort: component-import sprawl means some page labels come from imported components the audit only partially follows; treat 🟡 as a prompt to confirm, not a confirmed gap.
- BEHAVIOR_MISMATCH (doc says "click X to Y" but code shows X does Z) is a future enhancement — too low-precision for static v1.
- Runtime (Playwright against the deployed app) is a future enhancement; this v1 is static-only.

## Baseline interpretation (hand-annotated, 2026-06-01)

> Part B of ADR 049, redesigned (per Misha) from a backwards-facing completed-plan
> walker to this forward-facing page-vs-doc accuracy audit. This is the first
> baseline run.

**Headline.** Of the 28 live contractor-facing pages:

- **🔴 STALE: 0.** No doc references UI that has been removed/renamed — every
  bold UI term the docs name still exists in `src/`. The docs are *accurate*
  about what they describe today. This is the honest, correct baseline: the
  audit's value is catching **future** drift (a button rename, a removed
  section) the moment it happens, not flagging phantom problems now.
- **⚪ MISSING_DOC: 0.** Every contractor-facing page has a support doc.
- **🟡 UNDOCUMENTED: 73 page sections across 25 pages.** These are real,
  forward-facing coverage gaps: page section headings (e.g. `Funnel Status`,
  `Health Indicators`, `Quiet Hours`, `Weekly Trends`, `AI Intelligence`,
  `Communications`, `Reviews`) that exist on the page but the page's support
  doc never mentions. They are 🟡 (not 🔴) because an uncovered section is
  *less harmful* than a misleading one — it reduces doc utility but doesn't
  point a contractor at something that isn't there. These are the doc team's
  enrichment queue.

**What this replaces.** The originally-merged Part B walked completed plans
retrospectively and reported a page-doc *existence* check (28/28 documented).
That was backwards-facing and didn't answer the question that matters — *does
each live page's doc still describe what the page does now?* This audit does,
and it surfaces the per-section coverage gaps (the C-47 transfer flow and
Smart-Import depth gaps tracked as `COMPLETION-AUDIT-DEPTH-DOWNSTREAM-01` are
this same class — a page has a doc, but a shipped flow on it isn't covered).

**Precision note.** STALE is deliberately high-precision (a doc term is flagged
only if it appears NOWHERE in `src/`), so 0 STALE means 0 — not "the audit
missed them." An earlier draft using a "click X" prose extractor produced 18
false STALE from truncated sentence fragments; that extractor was removed in
favor of bold-term-only, label-shaped extraction. UNDOCUMENTED is heading-based
(button-label extraction produced ~224 noise hits and was dropped); treat each
🟡 as a confirm-and-cover prompt.

**Re-run:** `page-doc-accuracy-audit.sh --project <project-path> --out <path>`
(weekly via a scheduled task, or on-demand; a project may wire
`npm run audit:doc-accuracy` to call it). Exit 1 iff any STALE.
