### Assumptions

- **The orchestrator's "AFTER discovery-surfacer" + "BEFORE decision-context-pending-surfacer" instructions are simultaneously satisfiable** by placing the two new hooks contiguously in the order `discovery-surfacer.sh` → `decision-context-replay.sh` → `decision-context-pending-surfacer.sh` (replay before pending-surfacer so newly-flushed fallback-queue events are visible when the surfacer scans pending items, per the plan's documented dependency).
- **`de._category` is the correct marker for "this item came from a decision-context fence."** Confirmed by reading `adapters/claude-code/hooks/decision-context-gate.sh:495` (`var details = Object.assign({}, data, { _category: cat, surfaced_by: "decision-context-gate" });`) — the gate stamps `_category` on the `item-details-set` event's `details` payload, and that payload flows through `state.js` into the GUI's `it.details` object verbatim.
- **The fence schema in `decision-context-schema.d.ts` is the source of truth for field names.** I followed the TypeScript declarations exactly (e.g., `what_it_does` not `description`; `reply_with` not `phrase`; `the_ask` not `request`; `action_taken` not `summary`); cross-checked against `decision-context-schema.js` which uses identical Zod field names.
- **Pre-existing Walking Skeleton failure at HEAD `895b2cd` is out of scope for Task 9-full** and will be diagnosed separately by the orchestrator. The PR cover should call this out as a known regression worth investigating (likely an emit-hook path issue introduced by an earlier wave; not my scope to chase).
- **The legacy GUI assumes `de.recommendation` could be either a string OR an object with `{option_key, reasoning}`.** This is back-compat-preserving: the legacy code calls `detailRow('Recommendation', de.recommendation, projectKey)` which would coerce an object to `"[object Object]"` and render it via `linkifyDocs` — visually broken but not a crash. My change replaces the unconditional `add(detailRow(...))` with a `typeof`-branched render. Legacy items with string-shaped recommendations are unaffected.
- **The `det-chip-*` CSS classes will render readably even without my CSS additions** (graceful degradation) — they have explicit colour-coded backgrounds for urgency / reversibility / state, but the text inside is rendered with the default font and would still be legible if app.css wasn't loaded. The CSS additions are quality-of-life, not correctness-critical.
