# UX Design Principles

These apply to all UI work. Every user-facing state must guide the user to a resolution.

1. **Errors suggest a solution.** "Failed to save" → "Failed to save — check your connection and try again."
2. **Suggestions link directly to action.** "Click to categorize" must open the picker, not navigate away.
3. **Empty states explain why and offer a first action.** "No transactions" + "Import or add one manually" + button.
4. **Destructive actions require confirmation with reversibility info.** "Archive? All data preserved, restore anytime."
5. **Success states confirm the change and reveal what's next.** "Account archived. View archived accounts →"
6. **Loading states describe what's loading.** "Loading payment history..." not a bare spinner.
7. **Warnings by color.** Yellow = informational/fixable. Red = blocking/data risk. Never red for informational.
