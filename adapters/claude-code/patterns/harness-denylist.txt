# The codename below is also a generic electrical-engineering noun. Its
# pattern must catch the codename standalone (prose either case, file paths,
# identifiers, end-of-line) but must NOT catch the generic two-word idiom
# "<word> breaker" / "<word>-breaker" (nl-issue [47]: the old right boundary
# `(?:\s|$|[^A-Za-z])` false-blocked PR #91's spawn-cascade prose until the
# guards were renamed). POSIX ERE has no lookahead, so "not followed by
# 'breaker'" is spelled as the prefix-complement alternation below: after the
# word, allow (spaces/hyphens then EOL) | (any non-letter that is not a
# space/hyphen) | (spaces/hyphens then a word that is NOT 'breaker'-prefixed).
# The scanner greps -iE, and GNU grep folds bracket expressions under -i, so
# [^b] etc. also exclude 'B'. Accepted narrowings (proven in the scanner's
# --self-test, scenarios c1-c9): "<word>[ -]breaker..." never trips; the
# truncated forms "<word> b" / "<word> br" / ... / "<word> breake" at
# end-of-line never trip (six near-impossible strings, accepted).
(^|[^-A-Za-z0-9])Circuit([[:space:]-]*$|[^-A-Za-z[:space:]]|[[:space:]-]+([^b[:space:]-]|b[^r]|br[^e]|bre[^a]|brea[^k]|break[^e]|breake[^r]))
(^|[^-A-Za-z0-9])Foresight(?:\s|$|[^A-Za-z])

# === Cloud-bucket URLs with environment fragment ===
# S3/GCS bucket paths whose name embeds prod/dev/staging — these typically
# leak operational infrastructure references. Generic bucket URLs without
# an environment suffix are NOT matched (too high a false-positive rate
# in tutorials and docs).
s3://[a-z0-9-]+-(prod|dev|staging)/
gs://[a-z0-9-]+-(prod|dev|staging)/

# === OAuth client-id shapes ===
# Google OAuth client ID format: 12 digits, dash, 32 alphanum, .apps.googleusercontent.com
[0-9]{12}-[a-z0-9]{32}\.apps\.googleusercontent\.com
# GitHub App client ID format: Iv1. + 16 hex chars
Iv1\.[a-f0-9]{16}

# === Database connection strings with embedded credentials ===
# postgres://, mysql://, mongodb://, mongodb+srv:// with user:password@host
(postgres|mysql|mongodb)(\+srv)?://[^/[:space:]]+:[^@[:space:]]+@

# === Additional service keys ===
# SendGrid API key: SG. + 22+ chars + . + 43+ chars
SG\.[A-Za-z0-9_-]{22,}\.[A-Za-z0-9_-]{43,}
# Stripe restricted keys (live + test)
rk_(live|test)_[A-Za-z0-9]{20,}
