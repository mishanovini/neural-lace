# harness-denylist.txt
# Patterns that must NEVER appear in Neural Lace harness code.
# Enforced by adapters/claude-code/hooks/harness-hygiene-scan.sh at pre-commit time.
# Format: one pattern per line. # begins a comment. Regex allowed (POSIX extended).
# Do NOT add generic English words; this must be high-signal-only to avoid false positives.
#
# Patterns are case-insensitive; scanner uses `grep -iE`.

# === Personal identifiers (owner of this private dev environment) ===
# NOTE: standalone `misha` pattern removed — caused false-positive on
# `**Owner:** Misha` attribution line (user-approved attribution-only policy).
# The filesystem-path patterns below (/c/Users/misha/, /Users/misha/, etc.)
# still catch any dangerous lowercase `misha` uses.
mishanovini
MishaPT
jagguar7

# === Generic credential/PII shapes (catch future additions) ===
# NOTE: generic email matching was removed 2026-04-18. POSIX ERE does not
# support negative lookahead, so an `@(?!example\.com)` pattern silently
# failed to compile and either no-op'd or produced false positives. The
# targeted entries above (jagguar7, pipeline-test@pocket-technician) catch
# the known leaks; broader email matching risks more false positives than
# prevented leaks. Reintroduce with per-file exemptions if a real leak
# surfaces.
