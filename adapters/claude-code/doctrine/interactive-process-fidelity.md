# Interactive-process fidelity — compact
> Enforcement: Pattern — self-applied
> Applies: any multi-stage process whose stages exist to collect the operator's answers

When a process is interactive — its stages exist to elicit the operator's
answers, dispositions, or approvals (PRD intake, plan-time interface-impact
decisions, discovery-protocol irreversible dispositions) — carry-forward
context from an upstream conversation is a BRIEFING, never a substitute for the
operator's actual answer.

**Structure vs authority.** Structure is legitimately autonomous: scaffolding,
file conventions, formatting, drafting a proposed answer FROM the carry-forward
context. Authority is never autonomous: the operator's actual answer to a
stage's question, the disposition of an open question, an approval to proceed.
Carry-forward context can inform a proposal; it cannot BE the answer.

Draft the best proposal you can from context, then surface the stage's actual
question WITH the proposal attached, and WAIT for the operator's reply — even a
one-word confirmation is the real convergence signal a synthesized answer is
not. Self-closing an open question by assigning your own disposition because
context "implies" an answer is the exact failure this rule prevents.

If you cannot wait (the operator is unavailable), halt at that stage and name
the blocker concretely — do not synthesize and proceed. Continue any structural
work that doesn't depend on the unanswered authority question; batch the
authority questions for the operator.

Mechanical gates that check an artifact's SHAPE (e.g. section presence) do not
verify this — an artifact can pass every shape check while every "answer" in it
was self-synthesized. This is a distinct check you make yourself.
