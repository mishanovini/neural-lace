# Code Conventions — compact
> Enforcement: typecheck/lint in CI; else Pattern — self-applied. Sources: typescript, api-routes, database-migrations, documentation.
> Applies: `**/*.ts`/`.tsx`; `src/app/api/**`; `supabase/migrations/**` (or equivalent migrations dir); all files.

- **TypeScript:** `strict: true` everywhere. **No `any`** without an explicit justification comment. **`import type`** for type-only imports. No `@ts-ignore` / `as any` — fix the type error. Explicit return types on exported functions. No `console.log` in committed code. No `// eslint-disable` without justification.
- **API routes:** after writing, test it (`curl -s .../api/<route> | jq '.'`); document the FULL response shape in evidence — every field, every nested object. Auth-required routes: test with a valid session token AND confirm the route is in the auth middleware matcher (`middleware.ts`). **Supabase queries: verify RLS policies allow the query for the expected user role.** New column reference → confirm it exists via a live query first.
- **Database migrations:** before writing, check existing row count for the table — if populated, the migration MUST handle existing rows. **New NOT NULL columns: add a DEFAULT, OR add nullable → backfill → THEN alter to NOT NULL — never NOT NULL without a default on a populated table.** New tables: add RLS policies immediately, not in a follow-up; include in `seed_org_defaults` if org-scoped. After running: verify schema + check-nulls; for enum/check constraints, verify compliance. Use `ON CONFLICT` for idempotent seed inserts.
- **Documentation:** doc comments (JSDoc/TSDoc) on exported functions/components explaining what+why; inline comments on complex business logic explain why, not what. API routes document request/response shape, auth requirements, error codes. Use `@deprecated`, not lingering TODOs. Architecture changes update project docs (AGENTS.md/README) in the same PR. **Update `.env.example`** whenever a new environment variable is added.
