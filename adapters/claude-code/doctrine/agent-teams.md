# Agent Teams — compact
> Enforcement: teammate-spawn-validator.sh (PreToolUse Task|Agent), task-created-validator.sh (TaskCreated), task-completed-evidence-gate.sh (TaskCompleted), team-aware tool-call-budget.sh. Full: doctrine/agent-teams-full.md
> Applies: only when explicitly enabled — otherwise this is a no-op and `orchestrator-pattern` governs multi-agent work.

- Agent Teams is disabled by default. Enable via `~/.claude/local/agent-teams.config.json` (`"enabled": true`, defaults `force_in_process: true`, `worktree_mandatory_for_write: true`, `per_team_budget: true`) PLUS the env var `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1` (Claude Code v2.1.32+). With the flag off, spawns carrying `team_name` are rejected and the harness behaves exactly as a non-team install.
- Prefer **`orchestrator-pattern`** (Task-tool dispatch, lead-orchestrator + fresh-context builders) as the default multi-agent topology — more battle-tested, fewer upstream bugs. Reach for Agent Teams only when teammate-to-teammate direct messaging or shared-task-list negotiation is the load-bearing requirement.
- Five known upstream Anthropic bugs you opt into: inbox messages to lead deferred until turn-end (#50779 — don't design lead-side polling loops); pane-based teammates drop lifecycle events at the parent (#24175 — harness forces `force_in_process: true` to sidestep); bypassPermissions doesn't propagate to background-spawned teammates (#43736); Delegate Mode tool-stripping on first turn (#24073/#24307 — use Spawn-Before-Delegate: spawn with a minimal ack prompt, wait for idle, THEN send the real task via SendMessage); `teammate_name`/`team_name` absent from hook event input (#45329 — harness derives membership by joining session_id against team config).
- In-process teammates (the default) get full harness enforcement; pane-based teammates lose parent-visibility events — override `force_in_process: false` only for visual debugging, never production team work.
- Windows Git Bash has no tmux — in-process is structurally forced there anyway.
