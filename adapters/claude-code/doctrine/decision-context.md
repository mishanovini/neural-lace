# Decision-Context — compact
> Enforcement: decision-context-gate.sh (Stop, fence enforcement — RETIRING per D.4), decision-context-reply-emit.sh (UserPromptSubmit), decision-context-pending-surfacer.sh (SessionStart). Full: doctrine/decision-context-full.md
> Applies: every agent→operator surface that solicits a decision, asks a question, assigns an action item, or logs an autonomous action.

- **Operator-facing format is constitution §3's compact table** (operator-approved 2026-07-02) — one-sentence ask, ≤5-line context with links, options table (≤4 rows), "My pick," "Reply with." This REPLACES the `:::` fence Markdown grammar as what the operator sees. Use §3's format, not the fence, when writing a decision/question/action-item into a response.
- **D.4 retires the fence's Stop-hook enforcement** (`decision-context-gate.sh`'s Tier-1 hard-block + Tiered-Scan classifier) — superseded by the constitution §3 format being the operator surface. Decision: "decision-context enforcement retired (emit writers kept)" — **emit-side writers kept**: the code paths that land `decision-raised` / `question-raised` / `action-added` / `autonomous-action-logged` + `item-details-set` events into the workstreams state file via the sole-normative `decision-context-schema.js` module continue to fire — only the fence-grammar *authoring requirement on the prose* goes away, not the tree-tracking of the decision.
- Sole-normative validator: `workstreams-ui/state/decision-context-schema.js` (Zod). NO shell re-implementation, ever — both the hook (`node -e require(…)`) and the GUI consume this one module. If the schema needs to change, it changes there only.
- Four content categories the emit layer models: `decision` (options + tradeoffs + recommendation + reply_with), `question` (why_asking + what_ive_tried + answer_shape), `action_item_for_user` (the_ask + why_assigned + what_im_doing_meanwhile), `autonomous_action` (fait-accompli log — action_taken + reasoning + reversibility + ≥1 reference).
- Cross-field constraint the schema locks: if `expires_at` is set, `default_if_no_response` must be set too; for `decision`, the default must reference an option whose `reversibility_cost` is `free` or `cheap` — never auto-apply an expensive/irreversible default on expiry.
- Interim guidance while the fence gate is live and the operator wants the §3 format instead: write the gate's waiver rather than emit the format the operator rejected; the gate's fence enforcement retires in D.4 as designed.
